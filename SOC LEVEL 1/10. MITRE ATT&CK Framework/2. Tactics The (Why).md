Tactics represent the high-level goals of an adversary during a cyberattack. They answer the question: **"Why is the attacker performing this action?"** The MITRE ATT&CK Enterprise matrix organizes these goals chronologically, mimicking the phases of a real-world breach.

---
## 🏛️ The 14 Tactics of the Enterprise Matrix

Tactics are the structural pillars of the ATT&CK matrix.

[!IMPORTANT] **Matrix Scope:** The Enterprise MITRE ATT&CK framework currently contains: **14 tactics** arranged from left to right to map out the entire intrusion lifecycle.

```
[ Reconnaissance ] ──► [ Resource Dev ] ──► [ Initial Access ] ──► [ Execution ] ──► [ Persistence ]
        │
        ▼
[ Privilege Escalation ] ──► [ Defense Evasion ] ──► [ Credential Access ] ──► [ Discovery ]
        │
        ▼
[ Lateral Movement ] ──► [ Collection ] ──► [ Command & Control ] ──► [ Exfiltration ] ──► [ Impact ]
```

---
## 🔑 Key Tactical Phases & Case Studies

Each of the 14 tactics targets a specific adversary objective:

### 1. Persistence (TA0003)

When an attacker gains a foothold, their priority is establishing a method to return if the system reboots, user passwords are changed, or security patches are applied.

[!NOTE] **Surviving Reboots:** The tactic that focus specifically on maintaining access to a compromised environment across system restarts and credential changes is: **Persistence (TA0003)**. Common techniques include scheduled tasks, service creation, and registry run key insertions.

### 2. Credential Access (TA0006)

Attackers require user credentials to gain privileges and blend in with legitimate network actions.

[!WARNING] **Stealing Credentials:** The specific tactic that involves an adversary stealing passwords, hashes, and authentication tokens from system memory is: **Credential Access (TA0006)**. Mimikatz executions, keylogging, and credential dumping from LSASS memory belong to this tactical category.

## 📊 Summary of Tactic Alignments

|Tactic Name|Tactic Code|Goal Description|
|---|---|---|
|**Initial Access**|`TA0001`|How the attacker enters the network (e.g., Phishing).|
|**Execution**|`TA0002`|Running malicious code on a host (e.g., executing scripts).|
|**Defense Evasion**|`TA0005`|Avoiding detection (e.g., disabling Windows Defender).|
|**Lateral Movement**|`TA0008`|Moving from host to host (e.g., using RDP).|
|**Exfiltration**|`TA0010`|Stealing collected data (e.g., uploading to cloud storage).|
|**Impact**|`TA0040`|Disrupting or destroying services (e.g., deploying Ransomware).|
