MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) is the globally accessible, industry-standard knowledge base of threat actor behaviors. It has become the universal language for mapping detections, analyzing threat intelligence, and conducting security exercises.

---
## 🧭 Why MITRE ATT&CK is Unique

Traditional cybersecurity databases (like CVE or NVD) focus on documenting software flaws or vulnerabilities. MITRE ATT&CK takes a completely different approach.

[!IMPORTANT] **Adversary Behavior Focus:** MITRE ATT&CK catalogs: **Adversary behaviors and techniques based on real-world observations rather than software vulnerabilities**. Instead of mapping specific coding flaws, ATT&CK maps exactly what attackers do post-compromise (e.g., how they establish persistence, move laterally, or dump credentials) in order to achieve their goals.

---
## 🏛️ The Three Layers of the Framework

The Enterprise ATT&CK matrix is structured into a hierarchical taxonomy:


```
[ Tactics (Adversary Goals / "Why") ] (14 tactical areas arranged chronologically)
               │
               ▼
[ Techniques (Adversary Methods / "How") ] (e.g., T1566 Phishing)
               │
               ▼
[ Sub-techniques (Specific Variations) ] (e.g., T1566.001 Spearphishing Attachment)
               │
               ▼
[ Procedures (Real-world Implementations) ] (e.g., APT29 executing a specific macro payload)
```

[!NOTE] **Framework Hierarchy:** The three structural layers of the ATT&CK framework are: **Tactics (goals), Techniques (methods), and Procedures (real-world implementations)**.

---
## 🗣️ Establishing a Universal Security Language

Prior to the adoption of the ATT&CK framework, different vendors and security groups used confusing, overlapping jargon to describe identical activities.

[!TIP] **Communication Enhancement:** MITRE ATT&CK improves threat intelligence communication between security teams because: **It provides a universal vocabulary so different teams describe adversary behavior using the same terminology and IDs**.

By referencing standard technique IDs (like **T1059.001** for PowerShell scripting or **T1078** for Valid Accounts), blue teams, threat intelligence analysts, and purple team auditors can communicate threats clearly without ambiguity.
