If tactics represent the high-level goals of an attacker, **techniques** and **sub-techniques** represent the specific actions they take to achieve those goals. Techniques are the core building blocks used by detection engineers and SOC analysts to configure logging and alerts.

---
## 🏛️ Framework Hierarchy: Techniques & Sub-Techniques

The ATT&CK matrix uses a nested structure to allow analysts to categorize behaviors with both high-level groupings and granular details.

[!IMPORTANT] **Understanding Sub-techniques:** The distinction between techniques and sub-techniques is defined as: **A sub-technique is a more specific method under a main technique, providing granularity for precise detection and response**.

- **Technique:** A broad classification of an action (e.g., **T1059** - Command and Scripting Interpreter).

- **Sub-technique:** A highly specific variation (e.g., **T1059.001** - PowerShell).

[!NOTE] **Identifying T1059.001:** In the MITRE ATT&CK taxonomy, the specific identifier **T1059.001** maps to: **PowerShell execution as a sub-technique of Command and Scripting Interpreter**. PowerShell is the single most abused administrative scripting language in Windows environments, allowing fileless payload execution directly in volatile RAM.

---
## ⚙️ Techniques vs. Procedures

To understand real-world actions, analysts must separate general classifications from actual implementation scripts or tools.

```
       [ Technique ] (Broad definition: T1059.001 - PowerShell)
            │
            ▼
       [ Procedure ] (Real-world implementation: powershell.exe -nop -w hidden -enc ...)
```

[!WARNING] **Adversary Procedures:** In ATT&CK terminology, a **procedure** is: **A real-world implementation of a technique, such as a specific tool or command used by a threat actor**.

For instance, when a threat report states that the group _APT29_ executed a Base64-encoded command line calling a remote URL download script, they are describing a **procedure**.

## 🧰 Key Techniques Every SOC Analyst Must Know

- **T1053 — Scheduled Task/Job:** Configuring system tasks to execute commands at specific intervals (used for persistence).

- **T1547.001 — Registry Run Keys / Startup Folder:** Adding values to autostart registry paths to survive system reboots.

- **T1055 — Process Injection:** Injecting malicious code into the memory space of a running, benign process (e.g., `explorer.exe`) to hide execution.

- **T1003 — OS Credential Dumping:** Stealing passwords or hashes from volatile LSASS memory (e.g., via Mimikatz).

- **T1562 — Impair Defenses:** Actively disabling EDR agents, stopping logging services, or altering local firewall policies.
