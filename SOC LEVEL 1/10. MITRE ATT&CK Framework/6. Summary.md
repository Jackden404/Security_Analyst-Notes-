## 🏛️ Defining the MITRE ATT&CK Framework

MITRE ATT&CK provides the standard taxonomic vocabulary for documenting adversary behaviors.

[!IMPORTANT] **What ATT&CK Stands For:** The acronym stands for: **Adversarial Tactics, Techniques, and Common Knowledge**.

---
## 🧭 The Taxonomic Hierarchy

The matrix is structured to translate broad operational intentions into technical definitions.

### 1. Tactics (Columns)

Tactics describe the high-level objectives or goals of an attacker (e.g., _TA0003 - Persistence_).

- The Enterprise matrix currently contains: **14 tactics** arranged sequentially from left to right.
### 2. Techniques (Rows)

Techniques explain _how_ attackers seek to fulfill a tactic's goal.

- An ATT&CK technique describes: **How adversaries achieve their tactical goals, the specific methods they use**.

[!NOTE] **Technique ID T1059:** The core technique code **T1059** maps to: **Command and Scripting Interpreter**. Specific scripting languages (like PowerShell or Python) are nested beneath it as sub-techniques (e.g., `T1059.001` for PowerShell).

## 📊 Visualization & Gap Assessments

To manage detection capabilities and map operational security tools:

### ATT&CK Navigator

[!TIP] **Visual Layering:** The primary web-based visualizer provided by MITRE to annotate and track your detection coverage is the: **ATT&CK Navigator**.

### Gap Analysis

Comparing EDR and SIEM rules against the navigator grid exposes weaknesses.

[!WARNING] **Identifying Blind Spots:** Performing an ATT&CK gap analysis is designed to identify: **Techniques you cannot currently detect, which represent blind spots in your defense**. Resolving these gaps allows detection engineers to write robust rules before an active campaign targeting those techniques breaches the perimeter.
