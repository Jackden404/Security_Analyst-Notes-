# (IaaS/PaaS/SaaS)

Cloud computing shifts the operational architecture of modern enterprises. Understanding the differences between cloud service models is essential to identify the boundary lines of security responsibilities.

---
## 🍕 The Cloud Service Hierarchy (Pizza Analogy)

Security responsibilities scale based on the model deployed. We can compare these boundaries to dining models:


```
┌────────────────────────────────────────────────────────┐
│              Cloud Responsibility Scale                │
└────────────────────────────────────────────────────────┘
  [ On-Premises ] ──► [ IaaS ] ──► [ PaaS ] ──► [ SaaS ]
   (Make Pizza)      (Bake Pizza)  (Deliver)   (Restaurant)
   ◄── You Own ─────── Both Share ──────── Provider Owns ──►
```

- **On-Premises:** You manage the building, physical servers, hypervisor, host OS, applications, and storage.

- **Infrastructure as a Service (IaaS):** The provider manages physical security, networks, and virtualization hosts. You run virtual machines on top.

- **Platform as a Service (PaaS):** The provider manages the operating system, runtimes, and middleware. You host application code and databases.

- **Software as a Service (SaaS):** The provider manages the entire application stack. You simply manage user access and data classification.

---
## 🏗️ Technical Responsibilities by Model

### 1. Infrastructure as a Service (IaaS)

With IaaS (e.g., AWS EC2 instances, Azure VMs), you deploy a blank operating system template on the cloud provider's physical hardware.

[!IMPORTANT] **Operating System Security:** In the IaaS model, the entity responsible for patching the guest operating system (like Windows Server or Ubuntu) is: **The customer**. While the provider ensures the hypervisor runs, security groups, host firewall configurations, and local OS patches remain your responsibility.

### 2. Platform as a Service (PaaS)

PaaS (e.g., AWS Elastic Beanstalk, Azure App Service) abstracts the OS layer, meaning you cannot log on via SSH or install standard security EDR agents. Security monitoring shifts entirely to application code auditing and API gateway logs.

### 3. Software as a Service (SaaS)

SaaS (e.g., Microsoft 365, Google Workspace, Salesforce) provides direct access to software features via web browsers.

[!NOTE] **Application-Only Delivery:** The cloud service model that gives you access to just the application with no underlying infrastructure management is: **SaaS**.

[!WARNING] **SaaS Security Risk:** The primary security concern unique to SaaS deployments is: **Misconfigured access controls and data exposure**. Because the infrastructure is secure, SaaS breaches almost exclusively occur when users accidentally share documents publicly, misconfigure MFA controls, or leave open anonymous access gates.
