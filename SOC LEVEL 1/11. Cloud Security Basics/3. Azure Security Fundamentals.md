Microsoft Azure is the dominant enterprise cloud provider. With its deep integration into Entra ID (formerly Azure Active Directory) and Microsoft 365, securing Azure requires a thorough understanding of cloud-native auditing, identity containment, and secrets storage.

---

## 📡 Unified Auditing & Automation: Microsoft Sentinel

To monitor an entire Azure tenant, collect log events from disparate resource groups, and automate responses, security teams deploy a cloud SIEM.

[!IMPORTANT] **What is Microsoft Sentinel?** Microsoft Sentinel is: **A cloud-native SIEM and SOAR platform**.

It leverages Kusto Query Language (KQL) to search through Azure AD sign-in logs, host events, and network packets to detect indicators of attack and execute automated playbooks (SOAR) to contain threats.

---
## 👥 Identity Governance: Entra ID and PIM

Azure uses Entra ID as its core identity plane, which supports advanced risk-based policies (Conditional Access) and privilege management.

### Privileged Identity Management (PIM)

Permanent administrative accounts (e.g., Global Administrators) are high-value targets for attackers.

[!NOTE] **Just-in-Time Elevation:** Privileged Identity Management (PIM) provides: **Just-in-time admin access with time-limited elevation**.

Instead of keeping accounts permanently elevated, admins must request activation for a specific task (e.g., 2 hours), which automatically de-escalates their permissions when the time window expires.

## 🔑 Secure Secrets Management: Azure Key Vault

Hardcoding SQL database passwords, API tokens, or SSL certificates inside source code or system configurations leads to massive exposure risks.

[!CAUTION] **Managing Secrets:** The dedicated Azure service that manages and securely stores secrets, keys, and certificates is: **Key Vault**.

By utilizing Managed Identities, Azure Virtual Machines can fetch secrets from Key Vault at runtime without storing any authentication credentials locally in configuration files.
