Before network traffic can be analyzed, it must be captured and logged. Capturing data frames traveling across network segments is the first step in diagnosing anomalies and tracing attacker behaviors.

---
## 🔌 Traffic Mirroring: TAPs vs. SPAN Ports

To inspect packets, investigators must gain access to network data. This is typically achieved using one of two methods:

```
[ Active Network Path ] ───► [ Physical TAP ] ───► [ Exact Duplicate (No Packet Loss) ]
                                   │
                                   ▼
[ Managed Switch ] ───► [ SPAN Port Mirroring ] ───► [ CPU Dependent (Potential Drop) ]
```

### 1. SPAN Port (Switched Port Analyzer)

[!IMPORTANT] **What is SPAN?** A SPAN port is: **A switch port that mirrors traffic from other ports for monitoring**. It is a software-configured feature on managed switches that duplicates packet frames from target source ports and sends them to a dedicated analysis port.

[!CAUTION] **Overload Packet Loss:** Why might a SPAN port drop packets under heavy load? **The switch CPU prioritizes forwarding over mirroring**. Because the switch's primary operational function is to route and forward production traffic, its ASIC and CPU will dump the mirror buffer first if resources are constrained during high congestion periods.

### 2. Network TAP (Test Access Point)

A physical, hardware-based device spliced directly into the physical network cable. Passive TAPs copy all signals at the physical layer without relying on CPU processing, guaranteeing zero packet drops, even under maximum load.

## 💾 PCAP & Capture Filters

Captured packets are written to a standard file structure called **PCAP (Packet Capture)** or the modern **PCAPNG (PCAP Next Generation)**.

### Capture Filters (Berkeley Packet Filter - BPF)

Capturing full packet streams on high-speed networks can quickly fill disk drives (a 1 Gbps connection generates up to 10.8 GB of raw data every minute). To manage this storage:

[!TIP] **Capture Filter Advantage:** The primary advantage of using capture filters is: **They reduce file size by excluding unwanted traffic at capture time**. Unlike post-capture display filters, BPF filters run directly inside the kernel network stack, dropping unwanted packets before they are written to disk.

### Examples of BPF Capture Filters:

```bash
# Capture only DNS queries (UDP port 53)
tcpdump -i eth0 -w dns.pcap udp port 53

# Capture only traffic related to a specific database server, excluding SSH
tcpdump -i eth0 -w db.pcap host 10.0.5.42 and not port 22
```
