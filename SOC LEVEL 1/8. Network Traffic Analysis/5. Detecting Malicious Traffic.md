To detect malicious activity on a network, security analysts must understand the specific traffic patterns generated by threat actors during command-and-control (C2), lateral movement, and data exfiltration phases.

---
## 💓 Command & Control (C2) Beaconing

Malware installed inside a network periodically reaches out to the attacker's server to ask for new instructions. This "heartbeat" communication is known as a **C2 Beacon**.

[!IMPORTANT] **Beaconing Patterns:** The primary signal of C2 beaconing is: **Regular, periodic connections to the same destination at consistent intervals**. While normal human web traffic is bursty and erratic, a malware beacon behaves mechanically, connecting to its target (e.g., every 60 seconds or every 5 minutes, 24/7) with consistent packet payload sizes (typically small, 64 to 512 bytes).

---
## 📡 DNS Tunneling

DNS Tunneling is a method of bypass where malware encodes malicious payloads or exfiltrated data directly inside DNS queries.

[!WARNING] **Detecting DNS Tunneling:** Why is DNS tunneling difficult to detect from individual queries? **Each query looks normal; the signal is in aggregate volume and subdomain entropy**.

Because standard firewalls allow outbound DNS (port 53) without inspection, attackers hide data inside subdomain strings (e.g., `aGVsbG8gd29ybGQ.evil.com`). To catch this, analysts must look at the big picture: high volumes of queries to a single newly registered domain, coupled with high string randomness (entropy) in the subdomain labels.

---
## 🔀 Lateral Movement Indicators

Once an attacker gains a foothold, they seek to compromise other machines on the same local network. This is called **Lateral Movement**.

```
[ Compromised Host ] ───► [ Remote Host ]
   • SMB connection on Port 445
   • Named pipe creation (e.g., \pipe\psexecsvc)
   • Remote execution via Admin shares (C$)
```

[!TIP] **Detecting PsExec:** The pattern that indicates PsExec-based lateral movement is: **SMB connections followed by named pipe creation to remote hosts**.

PsExec (a Sysinternals administrative tool commonly abused by threat actors) relies on SMB (port 445) to connect to remote administrative shares (e.g., `C$`), create named pipes (such as `\pipe\psexec`), and install a temporary service to run commands.
