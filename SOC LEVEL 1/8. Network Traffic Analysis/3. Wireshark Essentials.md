Wireshark is the industry-standard graphic protocol analyzer. It allows security analysts to dissect packet captures (PCAP/PCAPNG), apply display filters, reconstruct sessions, and isolate indicators of compromise (IOCs).

---
## 🖥️ The Wireshark Interface

The Wireshark interface is divided into three functional panes:

1. **Packet List (Top):** Summarizes individual packets (No., Time, Source IP, Destination IP, Protocol, Length, Info).

2. **Packet Details (Middle):** Breaks down the selected packet by network layers (Ethernet, IPv4/IPv6, TCP/UDP, and Application layer payloads).

3. **Packet Bytes (Bottom):** Displays the raw hex and ASCII representations of the packet data.

---
## 🔍 Following Sessions: Reassembling Streams

Network communications are split into hundreds of individual TCP/IP packet segments. To view the data chronologically as a single readable stream:

[!IMPORTANT] **TCP Stream Reassembly:** In Wireshark, the **Follow TCP Stream** function: **Reassembles and displays the entire conversation between two endpoints**.

Right-clicking a packet and selecting _Follow > TCP Stream_ takes the fragmented packet chunks and presents them as a single readable exchange (e.g., an HTTP request header followed by the server's HTML response), stripping away TCP headers and sequence metadata.

## 🚦 Display Filters & Flag Analysis

Unlike capture filters, display filters hide packets from view _post-capture_ without deleting them from the underlying PCAP file.

### Dissecting the TCP Handshake

Understanding TCP flags is critical for locating port scans and raw socket handshakes.

[!NOTE] **Filtering SYN packets:** The display filter query `tcp.flags.syn == 1 && tcp.flags.ack == 0` filters the packet list to show: **Only SYN packets (new connection requests)**. This matches the first step of the standard TCP 3-way handshake, letting investigators track which ports are being probed by external hosts.

### Common Display Filters:

- `ip.addr == 10.0.5.42` — Displays all traffic to or from IP `10.0.5.42`.

- `http.response.code >= 400` — Filters HTTP traffic to show client/server errors.

- `!(arp or dns or icmp)` — Excludes noisy background protocol traffic.

## 🎨 Protocol Color Coding

Wireshark uses automatic color rules to help analysts scan packet lists visually:

- **Green:** Standard TCP traffic.

- **Purple:** Encrypted SSL/TLS traffic.

- **Black background with Red text:** Network errors, retransmissions, or malformed packets.

- **Light Blue:**

[!TIP] **DNS Protocol Color:** Wireshark automatically applies the color **Blue** to highlight: **DNS traffic** (UDP port 53), allowing analysts to quickly locate domain name query activity.
