TCP (Transmission Control Protocol) and HTTP (Hypertext Transfer Protocol) make up the majority of modern web traffic. Being able to dissect the TCP handshake, evaluate HTTP request headers, and carve files out of packets are essential skills for network analysts.

---
## 🤝 The TCP 3-Way Handshake

TCP is a connection-oriented transport protocol. Before any application data (such as an HTTP request) can be sent, the source and destination must complete a synchronized handshake:

```
  Client                                     Server
    │                                          │
    │ ─── 1. SYN (Initial Seq Number) ───────> │
    │                                          │
    │ <── 2. SYN-ACK (ACK client, send own) ───│
    │                                          │
    │ ─── 3. ACK (Confirm connection) ────────>│
    ▼                                          ▼
```

[!IMPORTANT] **Handshake Order:** The correct, sequential TCP connection handshake flow is: **SYN, SYN-ACK, ACK**.
### Critical TCP Flags to Watch:

- **SYN:** Synchronize. Proposes initial connection.

- **ACK:** Acknowledge. Confirms receipt of packets.

- **RST:** Reset. Abruptly terminates a connection. A surge of RST flags can indicate port scanning activity or host disconnection.

- **FIN:** Finish. Gracefully closes the connection.

## 🔌 Dissecting HTTP Request Headers

HTTP sends data in cleartext, making it easy to examine request headers inside Wireshark. Key header properties include:

- **HTTP Method:** `GET` (requesting data), `POST` (uploading data).

- **URI:** The specific resource path requested (e.g., `/login.php`, `/shell.php`).

- **Host Header:** The target domain (e.g., `infoseclabs.io`).
### ⚠️ Evaluating the User-Agent

The `User-Agent` header identifies the application or browser requesting the web page. Legitimate web browsers send standard strings (e.g., Chrome, Safari, Firefox details).

[!WARNING] **Script-Based HTTP Requests:** If you encounter a User-Agent header like `"python-requests/2.28.0"`, it indicates: **A Python script is making the HTTP request, not a browser**. Attackers write Python scripts or use command-line utilities (like `curl` or `wget`) to automate C2 queries, beaconing, or download payloads.

## 💾 Forensic File Carving from PCAPs

If an attacker downloads a malicious payload or exfiltrates document files over unencrypted HTTP, the full file is stored inside the PCAP packet payload.

[!TIP] **Carving Files in Wireshark:** To extract transferred files from an HTTP packet stream in Wireshark: Go to **File > Export Objects > HTTP**.

This lists all files (HTML, PNG, EXE, ZIP) transferred during the capture session, allowing you to save them directly to disk for further hashing or static/dynamic malware analysis.
