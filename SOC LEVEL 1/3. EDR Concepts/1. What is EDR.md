**Endpoint Detection and Response (EDR)** is a security technology that continuously monitors endpoint devices (workstations, servers, mobile devices) to detect and respond to cyber threats in real time.

If traditional antivirus (AV) is a security guard checking IDs at the lobby door, EDR is a network of security cameras, motion sensors, and security response teams monitoring behavior inside the entire building.

---
## 1. Traditional AV vs. EDR: The Core Difference

Consider an alert where `WINWORD.EXE` spawns `PowerShell.exe` with a base64-encoded command string, but the file hash of the Word document has zero detections in VirusTotal. Why does EDR catch this when AV does not?

[!IMPORTANT] **EDR detects malicious behavior (Word spawning encoded PowerShell) regardless of file hash, while AV only matches known signatures**. EDR records and correlates the behavior of processes in memory, allowing it to spot anomalies even if the malicious file itself is completely unknown to security databases.

---
## 2. Living-off-the-Land (LotL) Attacks

Modern attackers avoid downloading custom binaries or writing executable files to disk to stay under the radar of traditional signature engines. Instead, they perform **Living-off-the-Land (LotL) attacks**.

A LotL attack is a **living-off-the-land attack using legitimate system tools (such as `certutil.exe`, WMI, or `powershell.exe`); AV fails because no malicious file hash exists to match**.

```
  [ Attacker Command ] ────> [ certutil.exe ] ───> Downloads Payload to Memory
                                    │
                                    └───> Spawns Scheduled Task
                                                │
                                                └───> Zero Files Written to Disk
```

Because the operating system natively trusts these binaries, traditional AV scans see nothing wrong with their execution. EDR, however, flags the unusual parent-child relationships and unexpected arguments.

---
## 3. Telemetry EDR Collects in Real Time

An EDR agent monitors host activity at a granular level, gathering telemetry across several domains:

- **Process Activity**: Tracks process creations, terminations, command-line arguments, and process ancestry.

- **File Actions**: Monitors read, write, rename, and deletion events.

- **Registry Access**: Captures changes to persistence keys (e.g., Run keys under `HKLM`).

- **Network Activity**: Records socket connections, target IP addresses, ports, and DNS requests.

- **Memory Telemetry**: Scans for reflective DLL injection, API hooking, and thread manipulation.

---
## 4. Response & Containment Capabilities

When a host is actively compromised (e.g., beaconing to a known C2 server at `203.0.113.42` every 60 seconds), immediate containment is necessary to block lateral movement.

[!CAUTION] EDR platforms allow analysts to invoke **network isolation (containment) to prevent lateral movement while preserving forensic access**. This command cuts off the host's communication to the rest of the local network and internet, but keeps the EDR console agent connected so that analysts can remotely pull memory dumps, query processes, and run remediation scripts.
