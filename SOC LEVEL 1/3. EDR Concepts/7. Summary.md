# EDR Final Quiz Summary

This summary serves as your study guide for the **Endpoint Detection and Response (EDR)** final quiz. Use it to review the core mapping techniques, triage rules, and product selection parameters.

---
## 1. Mapping Attack Chains to MITRE ATT&CK

Understanding how process trees translate to the MITRE ATT&CK matrix is a core capability for security operations. Consider the following common attack tree: `OUTLOOK.EXE` -> `WINWORD.EXE` -> `cmd.exe` -> `powershell.exe -Enc` -> `rundll32.exe` (from temp) -> `svchost.exe` (from temp) -> discovery commands -> `certutil` download -> `schtasks` persistence.

This chain maps directly to these MITRE ATT&CK techniques:

[!IMPORTANT] **T1566.001 (Phishing) -> T1059.001 (PowerShell) -> T1218.011 (rundll32) -> T1036.005 (Masquerading) -> T1082 (System discovery) -> T1105 (Ingress tool transfer) -> T1053.005 (Scheduled task)**.

## 2. Platform Selection Criteria

When evaluating which endpoint security controls to procure, SOC and engineering constraints govern the metrics that matter most.

If a security team has limited resources (e.g., a small group of 4 analysts) and must monitor a hybrid environment (60% Windows and 40% Linux) with the primary goal of stopping advanced threats without overwhelming the team:

[!TIP] The most critical evaluation factor is **Detection accuracy and MITRE ATT&CK coverage, since minimizing false positives directly impacts analyst productivity and detection quality**. High false-positive rates quickly lead to alert fatigue, causing small teams to miss real incidents.

## 3. Host-Level Indicators of Compromise (IoCs)

When triaging alerts, analysts compare running services against known OS baselines.

Suppose you receive an alert for `svchost.exe` making outbound TCP network connections to `185.220.101.42:443`. Upon investigating the process telemetry, you find:

- It was spawned by `rundll32.exe`.

- It is executing out of `C:\Users\jsmith\AppData\`.

- It runs under the context of the active local user `jsmith` rather than `SYSTEM`.

[!CAUTION] In this scenario, there are **Four indicators: wrong parent (rundll32 instead of services.exe), wrong path (AppData instead of System32), wrong user context (user instead of SYSTEM), and suspicious outbound connection**. Spotting this combination of architectural anomalies allows the analyst to confidently confirm process masquerading and remote execution.
