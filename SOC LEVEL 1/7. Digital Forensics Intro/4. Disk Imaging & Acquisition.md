Forensic acquisition is the process of creating a bit-perfect duplicate of a digital storage medium. Standard file copies (like copying directories via Explorer or `cp`) only capture active, visible files, leaving behind deleted fragments, hidden volumes, and filesystem structures.

---
## 💾 Why We Image Disks

In digital forensics, investigators never operate directly on original media (such as hard drives, SSDs, or SD cards).

[!IMPORTANT] **Preserving original data:** You must image a disk before beginning analysis: **To preserve the original evidence**. Analysis changes system registry files, shifts metadata timestamps, and triggers background defragmentation. Operating on a forensic clone ensures that the original source remains physically untampered.

---
## 🧱 Physical vs. Logical Acquisition

Acquisition strategies vary in scope and depth depending on the case requirements:

- **Logical Acquisition:** Copies only active directories and files matching specific system patterns. Fast but misses deleted files.

- **Physical Acquisition:**

[!NOTE] **Bit-Stream Imaging:** A **Physical (bit-for-bit) image** is the acquisition method that captures deleted data and slack space.

It duplicates every single raw sector (both `0`s and `1`s) on the disk, including unallocated blocks and slack space where deleted user files, logs, and artifacts reside.

---
## 🔌 Write Blockers: The Integrity Shield

To protect the physical media during connection, investigators must use a write-blocking interface (either a hardware device or specialized OS software).

[!CAUTION] **Write Protection:** The primary benefit of using a write blocker is: **It prevents any modification to the original evidence**.

The write blocker acts as a one-way bridge: it allows read commands to pass to the forensic workstation but drops all write, delete, or formatting commands sent by the operating system.

---
## 🛠️ Common Forensic Imaging Formats

### 1. Raw / DD Format

A simple bit-for-bit dump of the physical drive, typically written via command line:

```bash
# dd physical imaging command
sudo dd if=/dev/sdb of=/evidence/image.raw bs=4M conv=noerror,sync status=progress
```

- **Pros:** Compatible with every forensic utility.
- **Cons:** No built-in metadata, no compression, and no integrated integrity verification.
### 2. Expert Witness Format (E01)

Developed by Guidance Software (now OpenText) for EnCase.

- **Built-in Metadata:** Logs examiner names, case numbers, and timestamps directly inside the file header.

- **Integrity Checks:** Uses embedded MD5/SHA-1 hashes to detect data corruption automatically.

- **Compression:** Reduces file size (often by 40-50%) without losing any data.
