The Windows operating system generates thousands of system records, cache folders, and log hives to optimize user experience. In digital forensics, investigators exploit these artifacts to reconstruct user actions, find executed malware, and track data transfer.

---
## ⚡ Execution Artifacts: Prefetch, Shimcache & Amcache

When a program is executed on Windows, the OS logs metadata about the binary in multiple locations to assist with future loads and compatibility.

### 1. Prefetch Files

Windows creates a prefetch file (ending in `.pf`) for applications to optimize memory allocation during execution.

- **Path:** `C:\Windows\Prefetch`

- **Forensic Value:**

[!IMPORTANT] **Execution Tracking:** The Windows artifact that shows which programs have been executed, their file paths, and how many times they ran is: **Prefetch files**. They also store the last 8 execution timestamps and a list of all loaded DLLs, proving runtime execution even if the attacker deleted the source binary.
### 2. Shimcache (AppCompatCache)

- **Registry Hive:** `HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\AppCompatCache`

- **Forensic Value:** Tracks file paths and metadata of executables loaded by the loader. Useful for verifying file presence.
### 3. Amcache

- **Path:** `C:\Windows\appcompat\Programs\Amcache.hve` (A standalone registry hive).

- **Forensic Value:**

    [!NOTE] **Cryptographic Hashes of Executables:** The forensic registry artifact that records the SHA-1 hash of executed programs is: **Amcache**. This hash allows investigators to verify the integrity of the ran binary against VirusTotal or other threat feeds.

## 🔌 Windows Event Logs

Windows logs system, security, and application logs in a centralized folder.

[!WARNING] **Event Log Storage:** Windows Event Logs (files ending in `.evtx`) are physically stored in: **C:\Windows\System32\winevt\Logs**.

### Critical Event IDs to Monitor:

- **Event ID 4624 (Security):** Successful user logon (includes logon type e.g., Type 3 Network, Type 10 RDP).

- **Event ID 4625 (Security):** Failed user logon.

- **Event ID 7045 (System):** A new service was installed (common malware persistence mechanism).

- **Event ID 4104 (PowerShell):** PowerShell Script Block Logging, capturing the complete un-obfuscated script code executed in memory.

## 🗂️ User Activity & Folder Browsing Artifacts

- **Recent Documents (`.lnk` files):** Stored in `C:\Users\<user>\AppData\Roaming\Microsoft\Windows\Recent`. Logs shortcut pointers to files recently opened, retaining metadata of original volumes even if files are deleted.

- **UserAssist:** Located in `NTUSER.DAT\Software\Microsoft\Windows\CurrentVersion\Explorer\UserAssist`. Tracks explorer-run programs (encoded using ROT-13).

- **ShellBags:** Logs folders browsed by a user, proving a user explored a directory structure even if the directory was subsequently deleted.
