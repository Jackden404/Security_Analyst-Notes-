File systems serve as the digital ledger of any operating system. By analyzing the structural database and metadata tracked by the file system, investigators can trace attacker actions, recover deleted assets, and uncover anti-forensic techniques.

---
## 🗄️ NTFS: Windows File System Architecture

The **New Technology File System (NTFS)** is the default file system for Windows environments. Its forensic focal points center around the MFT and data stream features.

### 1. The Master File Table (MFT)

At the core of NTFS is a hidden system file named `$MFT`.

[!IMPORTANT] **Understanding the MFT:** The Master File Table (MFT) is defined as: **A database of all file records on the volume**. Every single file and folder on the NTFS partition has at least one 1024-byte record in the MFT containing metadata (file name, size, timestamps, permission descriptors, and data block run offsets). Even when a file is deleted, its MFT entry is simply flagged as "inactive" rather than deleted, allowing investigators to reconstruct file history.

### 2. Alternate Data Streams (ADS)

NTFS supports multiple data forks for a single file record, where the default stream is `$DATA` but additional custom streams can be appended.

```
Visible File (secret.txt) ──► Default Stream ($DATA) ──► Text content
                          └──► Alternate Stream (secret.txt:hidden.exe) ──► Hidden payload
```

[!WARNING] **Alternate Data Streams (ADS):** Alternate Data Streams can be used for: **Hiding data attached to files that is invisible to most tools**.

Attackers can execute commands like `type payload.exe > calc.exe:hidden.exe` to bind a malicious script or binary behind a benign system application, bypassing standard directory sizing checks and basic antivirus scans.

## 📅 Timestamps, Metadata, and Anti-Forensics

File systems record chronological timestamps (MACB in NTFS, Inode change logs in Linux ext4) indicating when files are created, modified, or read.

### The Art of Timestomping

Attackers seek to mask their footprints by modifying file metadata records.

[!CAUTION] **Timestomping Definition:** **Modifying file timestamps to hide activity** is known as timestomping.

Attackers use utilities to copy timestamps from older, legitimate system files (like `ntdll.dll`) onto newly compiled malware files in order to defeat chronology timelines.

**How to Detect Timestomping:** Compare `$STANDARD_INFORMATION` timestamps against `$FILE_NAME` timestamps. Standard information is easy to modify via system APIs, but `$FILE_NAME` attributes are typically only modifiable by the OS kernel, creating a telling divergence if tampered.

## 🐧 ext4: Linux File System Architecture

In ext4, directories do not map directly to file content database blocks. Instead, they reference **inodes**.

- **Inode:** A data structure that houses all file metadata (owner UID, permissions, block pointers, mtime, atime, ctime) _except_ for the file name itself.

- **Data Block Pointers:** Extents point to direct data blocks on disk where the actual bytes reside.
