## 🔀 Core Forensic Workflow Rules

In digital forensics, following correct rules of evidence preservation is essential to ensure admissibility under legal cross-examination.

### The Golden Rule of Forensics

[!IMPORTANT] **Evidence Preservation:** The primary administrative baseline in any digital investigation is to: **Preserve the original evidence**. You never analyze the original media directly; instead, you operate on a bit-perfect forensic clone.

### Integrity Verification

[!TIP] **Using Hashes:** The main purpose of hashing a disk image is: **To verify the image has not been altered**. Cryptographic hashing (like SHA-256) matches the hash of the clone against the original device to confirm they are bit-for-bit identical.

## 🛠️ Evidence Acquisition & Volatility Priorities

### Volatile Memory Analysis

Volatile data (RAM) vanishes as soon as a computer is rebooted or shut down.

- The primary command-line tool commonly used for memory forensics is: **Volatility**.
### Write Blockers

To ensure no system commands write data back to the evidence media during acquisition:

[!CAUTION] **Write Blockers:** Using a write blocker prevents: **Writing to evidence, preserving its integrity**. It stops any write commands from reaching the source disk while allowing the host to read all sectors.

## 📂 Filesystems & Operating System Artifacts

### Alternate Data Streams (ADS)

NTFS (Windows) supports multiple data streams mapped to a single file record.

- The NTFS feature that allows data to be hidden inside a file (invisible to standard dir scans) is: **Alternate Data Streams (ADS)**.

### Prefetch Files

Windows tracks program executions to optimize loading.

- The Windows artifact that tracks program execution, along with file path and execution count, is: **Prefetch**.

## 🎓 Summary of Forensic Competencies

1. **Strict Volatility Ordering:** Collect CPU, process tables, and RAM (via Volatility) first, before capturing persistent disk files.

2. **Unbroken Chain of Custody:** Track every single evidence transfer on a physical Chain of Custody form to prevent legal exclusion in court.

3. **RCA and Artifact correlation:** Parse system artifacts (Prefetch, Amcache, Event Logs) to reconstruct a complete timeline of user and attacker activity.
