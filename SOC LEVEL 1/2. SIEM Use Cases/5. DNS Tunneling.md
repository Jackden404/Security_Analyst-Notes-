# Use Case 5: DNS Tunneling (Hard)

**DNS Tunneling** is a advanced data exfiltration and Command and Control (C2) technique. It abuses the DNS protocol by packaging non-DNS data into standard DNS queries and responses.

Since DNS is essential for internet routing and access, firewalls rarely block it, making it a highly covert and effective channel for bypassing traditional boundary controls.

---
## 1. Why Do Attackers Prefer DNS?

In almost every secure network architecture, strict firewall rules block direct outbound HTTP/HTTPS connections from sensitive internal zones. However, **firewalls almost always allow DNS (port 53) outbound, making it an effective bypass for network security controls**.

Attackers route their queries to their own external **Authoritative Name Server**, which receives and decodes the DNS requests.

---
## 2. Anatomy of a Tunneling Query

In a standard DNS query, subdomains are human-readable (e.g., `www.google.com`). In a DNS tunnel, **exfiltrated data or C2 commands are encoded (typically base64 or base32) in the subdomain field**.

```
  [ Encoded Data Payload ]                  [ Target Domain ]
┌───────────────────────────┐            ┌────────────────────┐
 SGVsbG8gV29ybGQgZnJvbSB... .subdomain.  attacker-domain[.]com
```

When the local DNS resolver receives this, it forwards the query to the root name servers, which eventually hand the query to the authoritative server for `attacker-domain[.]com`. The attacker's server extracts the subdomain prefix and decodes it back to raw files or shell commands.

---
## 3. Shannon Entropy Scoring

How can a SIEM distinguish legitimate DNS queries from tunneling payloads?

[!IMPORTANT] The statistical metric most effective for identifying DNS tunneling is a **Shannon entropy above 3.5 bits per character in subdomain labels, indicating encoded or encrypted data**.

Shannon entropy measures the randomness or information density of a string:

- **Low Entropy (< 3.0)**: Contains recognizable patterns and dictionary words (e.g., `mail`, `update`, `www`).

- **High Entropy (> 3.5)**: Contains encoded/encrypted random character sets, typical of base64 data streams (e.g., `ZG9jdW1lbnRzL3NlY3JldC5wZGY`).

---
## 4. Detection Query Example

SIEM alerts often count query length and evaluate character randomness. Below is an example of an extraction strategy:

|Metric|Threshold Alert Trigger|
|---|---|
|**Subdomain Label Length**|Exceeds 30 characters|
|**Shannon Entropy Score**|Exceeds **3.5 bits/char**|
|**Query Frequency**|Greater than 100 queries/min to a single domain|
|**Unique Subdomains**|High volume of unique subdomains resolved|

---
## 5. Investigation and Decoding Playbook

If a DNS tunneling alert fires:

1. **Extract the Queries**: Pull the full DNS query logs generated by the offending host targeting the suspicious domain.

2. **Attempt Decoding**: Try base64/base32 decoding the subdomain labels (e.g., stripping the domain and running `echo "SGVsbG8=" | base64 -d`) to inspect the contents of the payload.

3. **Containment**: Block the malicious external domain on internal DNS servers and isolate the compromised source machine immediately.
