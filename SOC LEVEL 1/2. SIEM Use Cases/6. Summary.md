# Module Summary: SIEM Use Cases

You have completed the **SIEM Use Cases** module. In this module, we covered five primary detection use cases that form the bedrock of daily SOC operations. Below is a comprehensive review guide for the final quiz.

---
## 1. Network Activity & Scanning Detections

Firewall and intrusion detection logs are the first line of defense against reconnaissance.

### Port Scanning

A firewall log showing rapid connections to many different ports on a single destination IP indicates a **Port scanning, where the attacker probes for open services on the target system**.

[!IMPORTANT] When port scanning is detected by network security controls, a well-configured firewall should **BLOCK the source IP and alert the SOC for investigation** to prevent the attacker from mapping out vulnerable network services.

## 2. Authentication Threats & Brute Force

When monitoring login logs, SOC analysts look for credential stuffing, brute force, and password spraying attempts.

### Windows Security Logs for Logins

- **Event 4625**: Indicates a failed login attempt.

- **Event 4624**: Windows Event ID representing a **successful login**.

### Spotting a Successful Compromise

In any credential attack scenario, a key indicator that the attack was successful is finding a **"Success" or "Accepted" authentication event (Event 4624) following a series of failed attempts**.

[!WARNING] Timing is critical when evaluating logins. For instance, a login occurring at **02:00 AM, an off-hours time that increases suspicion**, is a high-priority trigger because standard administrative tasks rarely occur at this time.

## 3. Host and Network Beaconing

Malware on infected systems must communicate back to its C2 infrastructure. Because it is programmed code, it behaves differently than human users.

### Identifying Beaconing

To detect malware beaconing in network traffic, analysts look for **regular, periodic time intervals between outbound connections to the same destination, indicating machine-precise communication**.

By calculating the standard deviation of these time deltas, we can immediately differentiate automated script beaconing (which has a standard deviation close to 0) from random human web browsing.

---
## 4. Advanced Threat Scenarios

### Impossible Travel

Stolen credentials can be spotted instantly when geo-IP analytics calculate that the physical travel time between two subsequent successful logins is physically impossible.

### DNS Tunneling

Attackers bypass strict boundary controls by encoding data in DNS subdomain queries. Because firewalls let port 53 (DNS) travel outbound without restriction, analysts must look for queries with high Shannon entropy scores (above 3.5 bits/char) to block these channels.
