# Open Source: The ELK Stack

The **ELK Stack** is the most widely deployed open-source log management and analytics platform in the world. ELK stands for **Elasticsearch**, **Logstash**, and **Kibana**. Together with **Beats**, these components form a complete log ingestion, storage, analysis, and visualization pipeline.

For organizations that cannot justify the cost of Splunk or Sentinel, the ELK Stack provides a powerful, highly customizable alternative at a fraction of the licensing cost.

---

```
   [ Endpoint / Server ]          [ Parser / Queue ]          [ Search DB ]        [ UI / Visualization ]
  ┌──────────────────────┐      ┌────────────────────┐      ┌──────────────┐      ┌─────────────────────┐

```

## 1. Elasticsearch: The Database & Search Engine

**Elasticsearch** is a distributed, JSON-based search and analytics engine built on Apache Lucene. It serves as the **database and search engine component of the ELK Stack**.
### Key Concepts

- **Indices**: Elasticsearch stores data in **indices** (analogous to databases in traditional SQL systems).

- **Shards & Replicas**: Each index is split into **shards** and distributed across multiple nodes in a cluster for horizontal scaling and high availability.

- **Mappings**: A mapping defines the schema of your documents (e.g., specifying which fields are IP addresses, timestamps, keywords, or full-text). Proper mapping is critical for query performance and data accuracy.

[!IMPORTANT] Elasticsearch uses **inverted indices to search through billions of log** rows in milliseconds, making it a high-performance backend database tailored for search queries rather than relational transactions.

## 2. Logstash: The Transformation Pipeline

**Logstash** is a server-side data processing pipeline that ingests data from multiple sources simultaneously, transforms/normalizes it, and sends it to your stash (usually Elasticsearch).

A Logstash configuration file is split into three main blocks:

```ruby
# Example Logstash Pipeline Configuration
input {
  syslog { 
    port => 514 
  }
  beats { 
    port => 5044 
  }
}

filter {
  grok { 
    match => { "message" => "%{COMBINEDAPACHELOG}" } 
  }
  geoip { 
    source => "clientip" 
  }
}

output {
  elasticsearch { 
    hosts => ["localhost:9200"] 
    index => "logstash-security-%{+YYYY.MM.dd}"
  }
}
```

[!TIP] Logstash is incredibly powerful but can be resource-heavy. In modern setups, many parsing filters are offloaded directly to Elasticsearch **Ingest Node Pipelines** to keep the footprint low.

## 3. Kibana: The Analyst Interface

**Kibana** is the visualization, exploration, and administration interface for Elasticsearch. It is the component of the ELK Stack that **provides visualization, dashboards, and the analyst interface**.

As a SOC analyst, Kibana is where you will execute all of your daily tasks:

|Interface Feature|Description|Security Use Case|
|---|---|---|
|**Discover**|Ad-hoc search, filtering, and analysis of raw events.|Investigating specific IP addresses or user activity.|
|**Dashboard**|Interactive collections of charts, graphs, and metrics.|Monitoring global authentication failures and system health.|
|**Lens**|Simplified drag-and-drop visualization builder.|Quickly creating new charts during an ongoing incident response.|
|**Alerting**|Rule-based notification system matching indexed data.|Routing critical alerts (e.g., Ransomware execution) to Slack/Webhooks.|
|**SIEM / Security app**|Tailored security interface built on top of indices.|Managing cases, alerts, and threat timelines.|

## 4. Beats: The Lightweight Shippers

**Beats** are **lightweight data shippers installed on endpoints to collect and forward logs** directly to Elasticsearch or Logstash. Unlike Logstash, which requires a JVM and substantial system resources, Beats are written in Go and run with a tiny CPU/RAM footprint.
### Common Beats Shippers

- **Filebeat**: Ships local log files (e.g., Syslog, auth.log, web server logs).
- **Winlogbeat**: Ships Windows Event Logs directly.
- **Metricbeat**: Ships system and service metrics (CPU, Memory, Disk).
- **Packetbeat**: Ships real-time network packet details.
- **Auditbeat**: Ships audit data and monitors file integrity (FIM).

[!NOTE] Filebeat comes with pre-configured **modules** for popular services (like Nginx, Apache, Active Directory, Cisco Firewalls) that package the parsers, index mappings, and Kibana dashboards automatically.

## Deployment Strategy & Cost Trade-Offs

When deploying an open-source SIEM pipeline, there are major trade-offs to consider compared to commercial platforms:

- **Financial Cost**: The ELK Stack is free to run under the Elastic License. You only pay for the underlying server infrastructure (cloud compute, storage, RAM) and the engineering hours required to manage it.

- **Operational Burden**: Unlike Splunk or Sentinel which are mostly managed, ELK requires you to handle cluster maintenance, shard allocation, index retention policies (using Index Lifecycle Management), and secure access controls (TLS/RBAC).
