
**Security Information and Event Management (SIEM)** collects security data and events, correlates them in real time, and manages the detection and response process. If the Security Operations Center (SOC) is the brain of security operations, the SIEM is the central nervous system.

---

## 1. Centralized Logging

In any modern network, log sources are everywhere: firewalls, endpoints, servers, databases, and cloud services. Without centralization, each source is an isolated island.

The SIEM resolves this by pulling logs from all of these separate systems into a single centralized repository. This provides SOC analysts with a single pane of glass to search logs, track investigations, and maintain system-wide visibility.

---
## 2. Ingestion & Endpoint Monitoring

To collect logs from across the network, specific components must sit on host machines:

- **A forwarder or agent installed on the endpoint**: Software installed on servers, laptops, and workstations that monitors local log files in real-time and forwards them securely to the central SIEM.

- **Syslog protocols**: Used to stream network device event data directly.

- **APIs**: Leveraged for SaaS and cloud platform log retrieval.

---
## 3. The Power of Correlation Rules

Storing logs is only the first step. The true magic of a SIEM lies in its **Correlation Engine**.

[!IMPORTANT] The primary function of correlation in a SIEM is **connecting related events across multiple sources to detect complex threats that individual events would not reveal**.

For example, a single failed login attempt is normal. However, 50 failed logins from a single external IP targeting 20 different user accounts within 3 minutes is a coordinated brute force attack. A correlation rule detects this pattern across logs, groups them, and flags it as a single high-priority alert.

---
## 4. Compliance and Log Retention

Beyond real-time threat hunting, SIEM platforms fulfill strict regulatory and compliance requirements (e.g., PCI DSS, HIPAA, SOX, GDPR).

[!NOTE] **Log retention** is critical because **compliance regulations require keeping logs for specific periods, and retention enables historical forensic investigation** during and after a security incident.
