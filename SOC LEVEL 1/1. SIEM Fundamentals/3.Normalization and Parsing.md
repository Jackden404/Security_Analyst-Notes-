
Raw logs come in dozens of formats. Syslog uses one structure, Windows Event Logs use another, cloud services use yet another, and every vendor invents their own custom fields. Normalization and parsing transform this chaos into a consistent, searchable, correlatable format.

---
## 1. What Normalization Achieves

[!IMPORTANT] The primary goal of log normalization is **converting different field names and formats from various sources into a single standard schema for cross-source correlation**.

Without normalization, you cannot easily correlate VPN credentials, Linux authentications, and web application requests because the field names differ across platforms:

- **Format Discrepancy**: If Windows calls a username field `AccountName` and Linux syslog calls it `User`, the SIEM **maps both to a single normalized field like user.name during the parsing and normalization process**.

Standard mappings usually include:

- `source.ip` — Source IP Address
- `destination.ip` — Destination IP Address
- `user.name` — Username
- `@timestamp` — Standardized Event Time
---
## 2. Field Extraction

Raw logs usually arrive as unstructured strings of text.

[!NOTE] **Field extraction using regex, grok patterns, or JSON parsing during ingestion** is the process that extracts structured fields like source IP, hostnames, process names, and destination ports from these raw text log messages.
### Example Transformation

- **Raw Log**:

    ```syslog
    Nov 3 14:22:05 web-server-01 nginx[4821]: 203.0.113.42 - - "GET /api/users HTTP/1.1" 200 1453
    ```

- **Extracted Structured JSON**:

    ```json
    {
      "timestamp": "Nov 3 14:22:05",
      "host.name": "web-server-01",
      "process.name": "nginx",
      "source.ip": "203.0.113.42",
      "http.method": "GET",
      "http.url": "/api/users",
      "http.status_code": 200,
      "http.bytes": 1453
    }
    ```

---
## 3. Timestamp Normalization

Every system format timestamps differently (e.g. Apache use `03/Nov/2025:14:22:05 -0500`, Unix use epoch timestamps like `1730658125`). The SIEM must convert all variations into a standardized, timezone-aware format (typically ISO 8601 UTC format, e.g. `2025-11-03T14:22:05Z`) to prevent correlation lag or rules dropping out of sequence.

---
## 4. Log Enrichment

Enrichment adds external contextual data to parsed log events during processing, turning a simple IP address or username into actionable security intelligence:

- **GeoIP Lookup**: Maps source/destination IPs to locations (e.g., Moscow, RU).

- **Threat Intelligence**: Feeds that flag known malicious IPs, command & control (C2) domains, or file hashes.

- **Asset Criticality**: Maps hostname metadata to identify department ownership or system classifications (e.g. Production Database vs. Guest Wi-Fi).
