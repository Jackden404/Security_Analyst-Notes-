## 🧭 The NIST SP 800-61 Standard

Incident response capabilities must align with recognized compliance frameworks.

[!IMPORTANT] **Core Framework:** The industry-standard document that defines the lifecycle of incident handling is: **NIST SP 800-61**.

---
## 🔀 The Phases of Incident Response

The incident handling cycle is structured into four sequential phases, starting from the beginning of the lifecycle to its completion.


```
       [ 1. Preparation ] ──► [ 2. Detection & Analysis ] ──► [ 3. Containment / Eradication / Recovery ] ──► [ 4. Post-Incident ]
```

### 1. Preparation Phase

[!NOTE] **Phase 1 Baseline:** The first phase of the NIST IR lifecycle is: **Preparation**. During this phase, you build your CSIRT capability, configure toolkits, set up communications, and define workflows before an event occurs.

### 2. Containment Phase

[!WARNING] **Isolating Threat Actors:** Isolating a compromised host from the network is part of the: **Containment** phase. This stops the lateral spread of malware or threat activities.

[!CAUTION] **RAM Capture Before Shutdown:** Prior to containing a host by severing its network or power connection, volatile memory must be preserved because: **It contains encryption keys, authentication tokens, command history, and malware artifacts lost on shutdown**.

## 🗂️ Playbooks & Post-Incident Action

### Playbooks

Organizations maintain distinct procedures for different threat categories (e.g., malware, web application breach, insider threat).

- A playbook is defined as: **A step-by-step procedure for handling a specific type of incident**.
### Post-Incident Activity

Learning from a security event is critical to prevent it from happening again.

- The post-incident activity phase consists of: **Lessons learned meetings, post-incident reporting, and process improvement recommendations**.

- This phase ensures recommendations have owners and deadlines to enhance the security posture of the organization.
