Eradication and Recovery represents the transition from defense to restoration. During this phase, security teams completely purge the threat from the network, hunt down persistent backdoors, verify system integrity, and safely return assets to production.

---
## 🚪 Root Cause Analysis (RCA)

Before you begin cleaning or rebuilding compromised assets, you must locate the vulnerability or vector used by the attacker to gain initial access.

[!IMPORTANT] **Complete Eradication Requirement:** To ensure the threat does not return, you must first: **The initial attack vector or root cause that allowed the compromise in the first place**. Simply deleting malware files (remediating symptoms) while leaving an open RDP port, unpatched VPN, or compromised Active Directory credential active is an invitation for the attacker to re-infect the network immediately.

---
## 🧹 Rebuild vs. Remediation Decision Matrix

When handling compromised operating systems, teams must decide whether to attempt manual cleanup (remediation) or wipe the disk and perform a fresh OS installation (rebuild).

|Rebuild (Wipe & Restore)|Remediation (Clean in Place)|
|---|---|
|🟢 **High Confidence:** Eliminates all hidden registry keys and modified system binaries.|🔴 **Higher Risk:** Can miss deep-seated persistence mechanisms.|
|⏱️ Takes time to spin up images and restore configurations.|⚡ Faster initial recovery, keeps system state alive.|

[!CAUTION] **When to Rebuild:** You should choose to rebuild a system rather than clean it: **When the attacker had root or SYSTEM access, the system handles sensitive data, or you cannot verify complete eradication**.

---
## ⚓ Hunting Persistence Mechanisms

Attackers set up "insurance policies" in case their main backdoor is deleted. Review these locations:

1. **Scheduled Tasks & Cron Jobs:** Tasks configured to run scripts or binaries on startup or defined intervals.

2. **Registry Run Keys:** Keys located in `HKLM\Software\Microsoft\Windows\CurrentVersion\Run` or `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`.

3. **WMI Event Subscriptions:** System event handlers that can execute arbitrary payloads.

4. **Web Shells:** Backdoor scripts left behind in directories hosted by IIS, Apache, or Nginx.

---
## 🔌 Reconnection Strategy: Gradual Reconnection

Once systems have been rebuilt or remediated and successfully verified (via vulnerability scanning and signature analysis), you must follow a controlled reintroduction path.

[!TIP] **Gradual Reconnection:** Bringing systems back online one at a time and monitoring them for 24-48 hours is preferred because: **It allows you to detect re-infection on individual systems before it can spread to the entire network**. This step-by-step process acts as a final validation gate, limiting the potential blast radius of any missed persistence vectors.
