Containment is the phase of incident response designed to limit the scope and blast radius of an active security compromise. Once an incident is identified, containment prevents the threat from spreading further and buying time for eradication.

---
## ⚖️ Short-Term vs. Long-Term Containment

Responders distinguish between two phases of containment depending on scope, timing, and operational requirements:

[!IMPORTANT] **Containment Strategies:** The distinction between containment methods is: **Short-term containment provides immediate, temporary protection to buy time, while long-term containment involves thorough, sustained measures to maintain security during full eradication**.

- **Short-Term Containment:** Immediate actions taken within minutes (e.g., disabling a compromised user account, blocking a single C2 IP address at the firewall, or isolating a workstation via EDR).

- **Long-Term Containment:** Sustained actions taken to protect the infrastructure while planning eradication (e.g., moving affected systems to a secure sandbox VLAN, applying core security patches, or placing additional network taps).

---
## 🔌 Host and Network Isolation Techniques

Security teams utilize multiple techniques to isolate compromised endpoints:

|Containment Method|Operational Action|Forensic Impact / Risk|
|---|---|---|
|**VLAN Quarantine**|Move host to a restricted, monitoring-only VLAN.|🟢 **Excellent:** Preserves volatile memory while preventing lateral spread.|
|**Switch Port Shutdown**|Disabling the physical switch port via command line.|🟡 **Moderate:** Completely cuts host off but prevents remote memory acquisition.|
|**DNS Sinkholing**|Redirecting C2 queries to a controlled internal IP.|🟢 **Excellent:** Disrupts C2 without alerting the attacker; captures telemetry.|

## 💾 Volatile Evidence Preservation (RAM)

A common mistake is turning off or rebooting an infected system immediately upon discovery.

[!CAUTION] **Preserving RAM:** Volatile memory must be captured prior to system shutdown because: **RAM contains encryption keys, authentication tokens, malware artifacts, and command history that are permanently lost on shutdown**. Removing power wipes this critical evidence, leaving analysts blind to fileless malware and active connections.

---
## 🤫 Silent Containment & Attack Awareness

If you identify an active threat inside your network, you must assess whether the attacker is aware of your discovery.

[!TIP] **Tactical Intelligence Gathering:** When an attacker is silently exfiltrating data and does not know they have been detected: **Implement monitoring and containment that does not alert the attacker, allowing you to observe and gather intelligence before taking visible action**. Moving too fast with visible blocks might alert the attacker, causing them to execute destructive payloads (e.g., ransomware) or switch to hidden backup persistence channels.
