An effective threat hunt requires high-fidelity data. If your logs are incomplete, your security team will operate with massive blind spots, allowing advanced persistent threat (APT) groups to remain undetected.

---
## 💻 Endpoint Telemetry: The Hunting Foundation

When hunting for active adversaries inside a corporate network, endpoint data provides the highest value because it logs the precise commands, execution paths, and dynamic files generated by an attacker.

[!IMPORTANT] **Most Valuable Data Source:** The primary data source considered the most valuable for threat hunting is: **Endpoint process execution logs (Sysmon/EDR)**.

- **Sysmon (System Monitor):** A free Microsoft utility that logs process creations with complete command line arguments (Event ID 1), network connections (Event ID 3), image loading (Event ID 7), and Alternate Data Stream creation (Event ID 15).

- **EDR (Endpoint Detection and Response):** Captures real-time behavioral telemetry, process ancestry trees, and memory injections.

---
## ⏳ Log Retention: The Time Window

Intruders often utilize "low and slow" techniques, staying dormant for weeks or months after establishing initial persistence before beginning active lateral movement or exfiltration.

[!CAUTION] **Why Retain Logs:** Log retention is critical for a threat hunting program because: **It determines how far back you can search for dormant threats**.

If your organization only retains logs for 30 days, you cannot hunt for an adversary who established persistence two months ago. A mature hunting posture requires a minimum of 90 days of "hot" searchable logs.

---
## 🗂️ Log Normalization: Cross-Correlation

A standard enterprise environment collects logs from thousands of different devices (Windows servers, Linux machines, proxy firewalls, cloud APIs, and identity providers), each writing logs in distinct formats.

[!TIP] **Unified Schema Value:** The primary benefit of normalizing log data to a common schema (such as OCSF or Splunk CIM) is: **It enables cross-source correlation and unified querying**.

Normalization maps varying field names (e.g., `src_ip`, `sourceIP`, `c-ip`) to a single standard field, allowing hunters to search for a specific indicator (like a compromised user ID or IP) across all databases with a single query.

---

## 📊 Priority Ranking of Log Sources

If your organization has budget constraints and can only ingest a subset of logs into the SIEM, prioritize them in this order:

1. **Process execution telemetry** (Sysmon/EDR).

2. **Authentication logs** (Active Directory, Azure AD, VPN authentication).

3. **DNS queries** (resolving domains).

4. **Network connection records** (Firewall/Proxy logs).

5. **Cloud APIs** (AWS CloudTrail, Azure Activity Log).
