## Why Correlation Matters

A single log entry rarely tells the whole story. Attackers operate in stages — reconnaissance, initial access, execution, persistence, exfiltration. Correlation rules link these stages together to detect the full attack chain.

## Multi-Event Correlation

Instead of alerting on a single event, correlation rules look for sequences of events:

**Example: Brute Force + Success**

- Event 1: 5+ failed login attempts within 5 minutes

- Event 2: Successful login from the same source within 10 minutes

- Correlation: Alert only when BOTH events occur

This dramatically reduces false positives compared to alerting on every failed login.
## Temporal Correlation

Time-based correlation looks for events that happen within specific time windows:

- **Burst detection**: 100+ events in 1 minute

- **Sequence detection**: Event A followed by Event B within 30 minutes

- **Dormancy detection**: Activity after a long period of silence

**Example:**

```
Event A: User logs in from New York at 9:00 AM
Event B: Same user logs in from London at 9:15 AM
→ Impossible Travel Alert (15 minutes is not enough to fly)
```

## Statistical Anomaly Detection

Instead of fixed thresholds, statistical methods learn what "normal" looks like:

- **Moving averages**: Compare current activity to the 30-day average

- **Standard deviation**: Flag events that fall outside 2-3 standard deviations

- **Seasonal patterns**: Account for time-of-day and day-of-week variations

**Example:**

- Normal: 50 SSH connections/hour

- Alert threshold: Mean + 3*SD = 150 connections/hour

- If you see 200 connections → Anomaly detected

## Tuning Correlation Rules

The key to effective correlation is reducing noise:

1. **Start broad** — Create rules that catch many potential threats

2. **Measure false positives** — Track how often alerts are benign

3. **Add context** — Include asset criticality, user behavior, threat intel

4. **Refine thresholds** — Adjust time windows and event counts

5. **Document everything** — Keep notes on why rules were changed
