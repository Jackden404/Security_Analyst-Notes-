## Insider Threat Detection

Insiders are dangerous because they have legitimate access. SIEM detects insider threats through behavioral indicators:

- Accessing files outside normal job scope
- Working unusual hours (3 AM access to sensitive data)
- Large data transfers to personal cloud storage
- Multiple failed access attempts followed by success

**Detection Rules:**

- Alert when a user accesses 10x more files than their baseline
- Flag downloads to USB devices or cloud storage
- Monitor for privilege escalation attempts

## Data Exfiltration Patterns

Attackers need to get data out. Common methods:

**DNS Exfiltration:** Large number of DNS queries to unusual domains, high-entropy subdomain names.

**HTTP/HTTPS Exfiltration:** Large uploads to external servers, encrypted traffic to unknown destinations.

**Physical Exfiltration:** USB device connections, print jobs to external printers.

## Lateral Movement Detection

Once attackers compromise one system, they move laterally:

- Pass-the-Hash: Using stolen password hashes
- RDP sessions between internal hosts
- SMB/WMI remote execution

**SIEM Detection:** Alert on RDP connections between workstations, flag WMI execution from unexpected sources.

## APT Indicators

APTs are patient, sophisticated attackers:

- Low and slow activities over weeks
- Living off the land (using legitimate tools maliciously)
- Encrypted C2 (command-and-control via HTTPS)
- Dormant implants that activate on schedule
