## S3 Bucket Exposure

Public S3 buckets are one of the most common cloud security issues:

**How it happens:**

- Accidental public ACL settings
- Overly permissive bucket policies
- Misconfigured static website hosting

**Detection:**

- AWS Config rules for public buckets
- S3 access logging
- CloudTrail monitoring for ACL changes

**Prevention:**

- Block Public Access at account level
- Use AWS Config rules to detect drift
- Enable S3 access logging
## IAM Privilege Escalation

Attackers escalate privileges through IAM:

**Common techniques:**

- Creating new admin users
- Attaching admin policies to existing roles
- Creating access keys for other users
- Modifying trust policies

**Detection:**

- Monitor IAM changes via CloudTrail
- Alert on policy attachments
- Track root account usage
## Security Group Misconfigurations

Common mistakes:

- 0.0.0.0/0 on sensitive ports (SSH, RDP)
- Overly permissive outbound rules
- Missing egress filtering
**Prevention:**

- Use AWS Config to detect drift
- Implement least-privilege network rules
- Regular security group audits

## Cloud Security Posture Management

CSPM tools continuously assess your cloud environment:

- Compare against benchmarks (CIS, AWS Well-Architected)
- Detect misconfigurations in real-time
- Provide remediation guidance
- Track compliance over time
