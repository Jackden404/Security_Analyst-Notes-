## Docker Security

Containers share the host kernel, which creates unique security challenges:

**Image Security:**

- Scan images for vulnerabilities before deployment
- Use minimal base images (Alpine, distroless)
- Never run containers as root
- Sign images with Docker Content Trust

**Runtime Security:**

- Monitor container behavior for anomalies
- Restrict container capabilities (drop unnecessary Linux capabilities)
- Use read-only file systems where possible
- Limit container resources (CPU, memory)

## Container Image Scanning

Scan images for known vulnerabilities:

- Trivy: Open-source, comprehensive scanning
- Snyk: Developer-friendly, integrates with CI/CD
- Aqua Security: Enterprise-grade scanning

**Key findings:**

- Critical/High vulnerabilities in base images
- Outdated packages with known CVEs
- Embedded secrets or credentials
- Malicious packages in dependencies

## Kubernetes Security

Kubernetes adds another layer of complexity:

- RBAC: Role-based access control for cluster resources
- Network Policies: Control pod-to-pod communication
- Pod Security Standards: Restrict container capabilities
- Secret Management: Never hardcode secrets in manifests

## Runtime Protection

- Falco: Open-source runtime security for containers
- Sysdig: Commercial container security platform
- Audit logging: Track all API calls to the cluster
