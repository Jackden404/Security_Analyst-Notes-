## Cloud Forensics Challenges

Cloud forensics differs from traditional forensics:

- You cannot physically seize the hardware
- Evidence is distributed across services
- Ephemeral resources may disappear
- You need API access, not physical access
## EBS Snapshot Analysis

EBS snapshots capture the state of an EC2 instance:

- Create snapshots before containment
- Mount snapshots as volumes for analysis
- Extract filesystem artifacts, logs, and memory dumps
- Use forensic tools (Volatility, Autopsy) on mounted snapshots
## VPC Flow Logs Analysis

VPC Flow Logs capture network traffic metadata:

- Source/destination IPs and ports
- Protocol and action (accept/reject)
- Bytes transferred

**Key analysis:**

- Unusual outbound connections
- Data exfiltration patterns
- Lateral movement within VPC
- Connections to known malicious IPs
## CloudTrail Investigation

CloudTrail provides the audit trail:

- Track all API calls made in the account
- Identify unauthorized access attempts
- Reconstruct attack timelines
- Correlate with other data sources
## Cloud Incident Response Process

1. **Detect** — GuardDuty, CloudTrail alerts
2. **Contain** — Isolate instances, revoke credentials
3. **Investigate** — Analyze logs, snapshots, flow logs
4. **Eradicate** — Remove malicious resources, patch vulnerabilities
5. **Recover** — Restore from clean snapshots
6. **Lessons Learned** — Update policies and controls
