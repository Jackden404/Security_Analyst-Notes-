## IAM Deep Dive
IAM is the foundation of AWS security. Every action in AWS goes through IAM. 

Key concepts:

**Policies:** JSON documents that define permissions. They can be attached to users, groups, or roles.
**Roles:** Temporary credentials for AWS services or federated users. Always prefer roles over long-term access keys.

**Least Privilege:** Only grant the minimum permissions needed. Use IAM Access Analyzer to identify over-permissive policies.
## Security Groups vs NACLs

**Security Groups:** Stateful firewalls at the instance level. If you allow inbound traffic, outbound is automatically allowed.
**NACLs:** Stateless firewalls at the subnet level. You must explicitly allow both inbound and outbound traffic.
**Best Practice:** Use Security Groups for instance-level control, NACLs for subnet-level defense.
## CloudTrail

CloudTrail logs every API call in your AWS account. It is your audit trail.

**What to monitor:**

- Root account activity (should be zero)
- IAM changes (privilege escalation)
- S3 bucket modifications (data exposure)
- Security group changes (network exposure)
## GuardDuty

GuardDuty is AWS threat detection service. It analyzes CloudTrail, VPC Flow Logs, and DNS logs to find threats.

**Key findings to watch:**

- Unauthorized access attempts
- Cryptocurrency mining activity
- Data exfiltration patterns
- Compromised credentials
