## From Detection to Response

EDR is not just about finding threats — it is about responding to them. Once you identify a compromised endpoint, you need to act quickly to contain the damage.

## Remote Isolation

The most powerful EDR response action is network isolation. This disconnects the endpoint from the network while still allowing the EDR agent to communicate.

When to isolate:

- Confirmed malware execution
- Active C2 communication detected
- Lateral movement in progress
- Data exfiltration detected

What happens during isolation:

- All network connections are blocked (except EDR agent)
- User can still work locally on the machine
- EDR console maintains full visibility

## Live Response Commands

EDR platforms let you execute commands remotely on compromised endpoints:

- **Kill process**: Stop a malicious process immediately
- **Delete file**: Remove malware artifacts
- **Run script**: Execute remediation scripts
- **Collect artifacts**: Gather evidence for investigation

Important: Always document what you do during live response — it may be needed for legal proceedings.

## Containment Strategy

When responding to an incident, follow this order:

1. **Identify** — Confirm the scope (which systems are affected?)
2. **Contain** — Isolate affected endpoints from the network
3. **Eradicate** — Remove malware and close the entry point
4. **Recover** — Restore systems to normal operation
5. **Lessons Learned** — Document what happened and improve defenses

## Common Mistakes

- Isolating too slowly (attacker may have already exfiltrated data)
- Isolating too aggressively (cutting off systems that need to stay online)
- Not preserving evidence before remediation
- Forgetting to close the initial access vector after cleaning up
