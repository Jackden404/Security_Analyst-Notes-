## Process Trees Explained

Every process on a system has a parent. When you launch Chrome, Chrome is the child of explorer.exe. When Chrome spawns a tab, that tab is a child of Chrome. This chain of parent-child relationships forms a process tree.

## Why Process Trees Matter for Security

Attackers often misuse legitimate processes. By understanding what parent-child relationships should look like, you can spot anomalies.

**Normal Examples:**

- winword.exe → outlook.exe (Word document opened from email)
- explorer.exe → chrome.exe (User launched browser)
- services.exe → svchost.exe (Windows services)

**Suspicious Examples:**

- outlook.exe → powershell.exe (Email spawned PowerShell)
- chrome.exe → cmd.exe (Browser spawned command shell)
- svchost.exe → whoami.exe (Service running recon commands)
## Key Anomalies to Detect

**Unusual Parents:** A process whose parent does not normally spawn children. For example, iexplore.exe (Internet Explorer) spawning cmd.exe is suspicious.

**Command-line Red Flags:**

- Encoded PowerShell commands (Base64 in command line)
- Download cradles (Invoke-WebRequest, certutil)
- Credential access tools (mimikatz, laZagne)

**Timing Anomalies:** Processes spawning at unusual times (3 AM) or in unusual sequences.

## Building Detection Rules

Create rules that alert on:

1. Office applications spawning shells (Word → PowerShell)
2. Service processes running user-level tools
3. Parent processes with unexpected child processes
4. Encoded or obfuscated command lines

## Real-World Investigation

An analyst sees explorer.exe → mshta.exe → powershell.exe in the process tree. HTA files are rarely used legitimately. The PowerShell command contains an encoded string that decodes to a download cradle. This process tree alone tells you this is a targeted attack using a phishing document.
