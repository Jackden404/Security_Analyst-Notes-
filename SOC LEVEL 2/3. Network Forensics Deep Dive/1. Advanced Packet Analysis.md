## Beyond Basic Packet Analysis

In Level 1 you learned to read packets. Now we go deeper — understanding what happens when protocols get complex, when traffic is encrypted, and when attackers try to hide.

## Protocol Dissection Beyond Basics

Every network protocol has layers. When you capture a packet, you see Ethernet, IP, TCP/UDP, and then the application layer (HTTP, DNS, etc.). Advanced analysis means understanding what each layer reveals.

**TCP Stream Reconstruction:** Individual packets are fragments. TCP stream reconstruction reassembles them into complete conversations. This lets you see the full HTTP request, the complete DNS query, or the entire SMTP session.

**JA3/JA3S Fingerprinting:** TLS encryption hides content, but the TLS handshake itself reveals information. JA3 hashes the client hello parameters (cipher suites, extensions) to create a fingerprint. Malware often uses unusual TLS configurations that JA3 can identify.

## TLS/SSL Inspection

Modern traffic is 90%+ encrypted. You cannot just read HTTPS traffic. Options include:

**Passive SSL Inspection:**

- Decrypt only the SNI (Server Name Indication) field
- See which domains are being accessed
- Cannot see the actual content

**Active SSL Inspection (MITM):**

- Proxy intercepts and decrypts traffi
- Re-encrypts before sending to destination
- Gives full visibility but has legal/privacy implications

**Certificate Transparency Logs:**

- Public logs of all TLS certificates issued
- Can detect malicious domains using legitimate certificates
## Encrypted Traffic Analysis

When you cannot decrypt traffic, analyze what you can see:

- **Flow data**: Source/destination IPs, ports, bytes transferred
- **Timing patterns**: Regular beaconing vs bursty traffic
- **SNI analysis**: Which domains are being accessed
- **Certificate analysis**: Who issued the certificate, when it was created
## Protocol Anomalies

Strange protocol behavior often indicates attacks:

- **DNS over HTTPS (DoH) to unusual resolvers**: Could be exfiltration
- **HTTP on non-standard ports**: Might be C2 communication
- **Long-lived TLS connections**: Possible persistent backdoor
- **Unusual DNS query patterns**: High entropy subdomains = data exfiltration
