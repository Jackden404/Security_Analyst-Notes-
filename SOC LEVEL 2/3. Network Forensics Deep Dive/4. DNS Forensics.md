## DNS: The Hidden Attack Surface

DNS is one of the most abused protocols in cybersecurity. Attackers use it for C2 communication, data exfiltration, and domain generation algorithms (DGAs).

## DNS Tunneling Deep Dive

DNS tunneling encodes data within DNS queries and responses. The attacker registers a domain, installs a nameserver, and encodes stolen data in DNS queries.

**How it works:**

1. Malware encodes data as a subdomain: aGVsbG8gd29ybGQ.evil.com
2. DNS resolver forwards the query to the attacker nameserver
3. Attacker extracts the encoded data
4. Response can carry commands back to the malware

**Detection:** Look for unusually long DNS queries, high-entropy subdomains, and excessive DNS traffic to a single domain.

## DNS Cache Poisoning

Attackers corrupt DNS cache entries to redirect users to malicious sites. This is devastating because it affects all users of a DNS resolver.

**How it works:**

1. Attacker sends spoofed DNS responses to a resolver
2. Resolver caches the false mapping
3. All users of that resolver are redirected

**Detection:** Monitor for unexpected DNS response patterns, sudden changes in domain-to-IP mappings.

## Domain Generation Algorithms (DGAs)

Malware uses DGAs to generate thousands of possible C2 domain names. The attacker only needs to register one of them. This makes blocking C2 domains extremely difficult.

**Detection:** Statistical analysis of DNS query patterns, entropy analysis of domain names, comparison against known DGA families.

## DNS over HTTPS (DoH) Challenges

DoH encrypts DNS queries, making traditional DNS monitoring impossible. Attackers love this because it hides their C2 traffic.

**Detection:** Monitor for DoH usage to unusual resolvers, analyze SNI patterns, look for TLS certificate anomalies.
