## Beacon Detection (C2)

Command-and-Control (C2) servers communicate with compromised systems using regular intervals — known as beaconing. Detecting these beacons is critical for finding hidden malware.

**How Beaconing Works:** Malware on a compromised system periodically checks in with the C2 server. This creates a pattern: regular DNS queries or HTTP requests at consistent intervals (every 5 minutes, every hour, etc.).

**Detection Methods:**

- Time-series analysis of DNS/HTTP requests
- Regularity scoring (low jitter = suspicious)
- Destination IP analysis (newly registered domains)
- Traffic volume analysis (small but consistent payloads)
## Data Exfiltration Patterns

Attackers need to get data out of your network. Watch for:

**Volume Anomalies:**

- Large uploads to external servers
- Unusual data transfer during off-hours
- Encrypted traffic to unknown destinations

**Protocol Abuse:**

- DNS queries with unusually long subdomains (DNS tunneling)
- Data hidden in HTTP headers or cookies
- Steganography (data hidden in images)

**Timing Patterns:**

- Regular small transfers that add up (low-and-slow exfiltration)
- Large transfers during maintenance windows
- Activity after hours of dormancy

## Port Scanning Detection

Port scanning is reconnaissance — attackers map your network before attacking.

**Types of Scans:**

- SYN scan (stealth scan): Most common, fast
- Connect scan: Full TCP connection, easily detected
- UDP scan: Slower but finds UDP services

**Detection:**

- Alert when a single source scans multiple ports on multiple hosts
- Monitor for connection attempts to closed ports
- Track new services appearing on unexpected ports

## Network Baseline Deviations

Establish what normal looks like, then alert on deviations:

- **Traffic volume**: 10x normal to a single destination
- **Protocol mix**: New protocols appearing on the network
- **Timing**: Activity during normally quiet periods
- **Geography**: Connections to unusual countries
