## What is Lateral Movement?

After compromising one system, attackers need to reach their target. Lateral movement is the process of moving from one compromised system to another within the same network.

## Common Lateral Movement Techniques

**Pass-the-Hash (PtH):** Attackers steal password hashes from one system and use them to authenticate to other systems without knowing the actual password. This is devastating because it works even when the user has a strong password.

**RDP Session Hijacking:** Attackers use stolen credentials to establish RDP connections to other systems. Look for RDP connections between workstations (not just from admin jump boxes).

**WMI/PowerShell Remoting:** Attackers use Windows Management Instrumentation or PowerShell to execute commands on remote systems. This is particularly dangerous because these tools are legitimate and widely used.

**SMB Lateral Movement:** Attackers use SMB file shares to move tools and scripts between compromised systems.

## Detecting Lateral Movement

**Network-based indicators:**

- RDP connections between workstations (not normal)
- WMI connections from unexpected sources
- SMB traffic spikes between internal hosts
- New service installations on remote systems

**Host-based indicators:**

- New user sessions appearing on systems
- Unusual authentication patterns
- Tools appearing on multiple systems
- Scheduled tasks created remotely
## Building Detection Rules

Alert on:

1. RDP connections from non-admin workstations
2. WMI execution from unexpected parent processes
3. Service installations on remote systems
4. Authentication from unusual source IPs
## Investigation Approach

When you suspect lateral movement:

1. Map all systems the attacker has touched
2. Identify the technique used (PtH, RDP, WMI)
3. Determine what data was accessed on each system
4. Check for persistence mechanisms on each system
5. Assess the full scope of the breach
