## Network Containment Strategies

When a network incident occurs, you need to contain the threat without disrupting business operations.

**Network Segmentation:** If your network is properly segmented, you can isolate affected VLANs while keeping other segments operational. This is why segmentation is so critical.

**Firewall Rules:** 紧急情况下，you can add firewall rules to block specific IPs or traffic types. But be careful — overly aggressive rules can break legitimate traffic.

**DNS Sinkholing:** Redirect malicious domain queries to a controlled server. This prevents malware from communicating with its C2 server without affecting legitimate traffic.

## Packet Capture Preservation

During an incident, preserving network evidence is critical:

1. **Start captures immediately** — You cannot go back in time
2. **Capture at multiple points** — Network tap, firewall, host
3. **Use sufficient buffer** — At least 1 hour of full capture
4. **Preserve DNS logs** — Often the most valuable evidence
5. **Document everything** — Timestamps, decisions, actions

## Network Isolation Techniques

**VLAN Isolation:** Move compromised systems to a quarantine VLAN. They can still communicate with each other (for investigation) but not with the rest of the network.

**Port Security:** Disable switch ports for compromised systems. Quick but disruptive.

**802.1X Re-authentication:** Force re-authentication for network access. Can be used to kick compromised systems off the network.

## Post-Incident Network Analysis

After containment, analyze the network to understand the full scope:

- Review all traffic to/from compromised systems
- Identify any other systems that may be compromised
- Check for persistence mechanisms (scheduled tasks, services)
- Review authentication logs for unauthorized access
- Update firewall rules to prevent recurrence
